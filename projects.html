Project 1 - Setting Up EJBCA In Your Home Lab

This guide will walk you through setting up EJBCA Community Edition in your home lab environment using Docker.

To begin, ensure you have:

  • - Docker installed on your system
  • - Sufficient resources (minimum 2GB RAM)
  • - Network access for downloading Docker images
  • - Administrative privileges
  1. If you meet the above requirements, follow these instructions:
  2. 1. Pull the EJBCA Docker image using:
    3. sudo docker pull keyfactor/ejbca-ce
  3. 2. Run the EJBCA Community Edition container with proper port mapping and hostname:
    4. sudo docker run -d -p 80:8080 -p 443:8443 -h localhost -e TLS_SETUP_ENABLED="true" keyfactor/ejbca-ce
  4. 3. Check if the container is running properly with:
    5. sudo docker ps
  5. 4. Retrieve the container logs to find the auto-generated SuperAdmin password using:
    6. sudo docker logs c0d4fdad387b
  6. 5. Open the ejbca.log file and search for the SuperAdmin password.
  7. 6. Navigate to the EJBCA RA interface in your web browser at:
    8. https://192.168.31.1/ejbca/ra/enrollwithusername.xhtml (replace 192.168.31.1 with your actual server IP address).
  8. 7. Login with username superadmin and the password found in the logs to generate a SuperAdmin certificate with RSA key algorithm, 2048 bits key size, and an appropriate certificate profile for admin access.
  9. 8. After generating the certificate, install it in your browser's Personal Certificate Store.
  10. 9. Finally, navigate to the EJBCA Admin Web interface at:
    11. https://192.168.31.1/ejbca/adminweb/, select the newly generated SuperAdmin certificate, and gain full administrative access to EJBCA.
  11. 10. Wowww!!! Congratulations, you have successfully set up EJBCA in your home lab!!!

Project 2 - Issuing a Client Authentication Certificate using EJBCA

This guide will how you how to issue a Client Authentication Certificate using your newly deployed EJBCA.

  1. Please follow the below instructions:
  2. 1. Open your browser and go to your EJBCA:
    3. https://192.168.31.1/ejbca/adminweb/
  3. 2. Log in with your admin credentials. (Use the SuperAdmin Certificate from the previous lab)
  4. 3. Create a ClientAuth Certificate Profile:
    5. EJBCA > CA Functions > Certificate Profiles > ENDUSER > Clone > ClientAuth > Create from template.
  5. 4. Edit the new ClientAuth Certificate Profile:
    6. Set Validity or End Date to 1y (1 year) > Check Client Authentication > Save.
  6. 5. Create a ClientAuth End Entity Profile:
    7. EJBCA > RA Functions > Manage End Entity Profiles > Add End Entity Profile > Name it ClientAuth > Add Profile.
  7. 6. Configure the new End Entity Profile:
    8. Subject DN Attributes > Add > emailAddress > Default Certificate Profile to ClientAuth (unselect others) > Set Default Token to P12 file > Save.
  8. 7. Add a New End Entity:
    9. EJBCA > RA Functions > Add End Entity > Choose a unique username > Set a password for certificate pickup > Use data from the e-mail address field.
  9. 8. Pick Up the Certificate at:
    10. https://192.168.31.1/ejbca/ra/#loaded > Enroll > Enter the username and password you set earlier > Select 2048 (key size) > Download P12 to get your certificate file.
  10. 9. Remember to download the CA Certificate too:
    11. Go to CA Certificates and CRLs in EJBCA > Download the CA Certificate in PEM format.
  11. 10. Finally, import the Certificate into Your Browser:
    12. Browser Settings > Security > Manage Certificates > Import > Select the downloaded P12 file > Enter the Enrollment Password.
  12. 11. Wowww!!! Congratulations, you have successfully issued and installed your Client Authentication Certificate.

Project 3 - Creating a Root Certificate Authority (Root CA) in EJBCA

This guide will how you how to create a Root Certificate Authority (Root CA) in your newly deployed EJBCA.

  1. Please follow the below instructions:
  2. 1. Access EJBCA Admin Web:
    3. https://192.168.31.1/ejbca/adminweb/
  3. 2. Log in with your admin credentials. (Use the SuperAdmin Certificate from the previous lab)
  4. 3. Create a Certificate Profile for the Root CA:
    5. EJBCA > CA Functions > Certificate Profiles > ROOTCA > Clone > Name it MyRootCAProfile > Create from template.
  5. 4. Edit the New Certificate Profile:
    6. Key Algorithm to RSA > Key Length to 4096 > Validity to 30y > Disable Authority Key ID, Subject Alternative Name, Issuer Alternative Name, LDAP DN Order > Save the profile.
  6. 5. Create a Crypto Token:
    7. EJBCA > CA Functions > Crypto Tokens > New > MyFirstRootCACryptoToken > Type to SOFT HSM for home lab; use HSM for production > Authentication Code (write this down!) > Create 2048 key pair > Save the Crypto Token.
  7. 6. Create the Root CA:
    8. EJBCA > CA Functions > Certificate Authorities > Add CA > MyFirstRootCA > Create > Select the CA settings from earlier > Create Root CA.
  8. 7. Wowww!!! Congratulations, you have successfully created your first Root Certificate Authority in EJBCA.
